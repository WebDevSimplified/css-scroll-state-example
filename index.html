The Future of CSS: Scroll-Driven Features

January 1, 2026 · 5 min read

Introduction

CSS is evolving rapidly, and some of the most exciting new features are centered around scroll-driven interactions. These features allow developers to create rich, interactive experiences without writing a single line of JavaScript.

In this article, we'll explore three powerful new CSS capabilities: ::scroll-button(), scroll-state() queries, and ::scroll-marker. Together, these features enable native carousels, sticky header effects, and intelligent navigation highlighting.

The best part? All of these features work with the browser's native scroll behavior, ensuring smooth performance and accessibility out of the box.

Scroll Buttons

The ::scroll-button() pseudo-element creates native navigation buttons for scroll containers. These buttons automatically handle scrolling by one "page" in the specified direction.

The carousel above uses ::scroll-button(left) and ::scroll-button(right) for the arrow navigation. The buttons automatically disable when you reach the start or end of the content.

Scroll State Queries

Container scroll-state queries let you style elements based on their scroll container's state. There are three types:

  • scrollable - Is there content to scroll to in a direction?
  • snapped - Is an element currently snapped to its container?
  • stuck - Is a sticky element currently stuck?

Notice the header at the top of this page? When you scroll down, it changes from transparent to a solid background with a shadow. This is achieved using scroll-state(stuck: top).

Similarly, the "back to top" button in the bottom-right corner only appears when you've scrolled down. This uses scroll-state(scrollable: top) to detect when the page can be scrolled upward.

These queries eliminate the need for JavaScript scroll event listeners, improving performance and reducing code complexity.

Scroll Markers

The ::scroll-marker pseudo-element generates navigation markers for scroll snap targets. Combined with scroll-marker-group, you can create dot navigation for carousels without any HTML markup.

The carousel above uses scroll markers for its dot indicators. Each slide generates a marker, and the :target-current pseudo-class highlights which slide is currently visible.

For more complex navigation needs, you can use scroll-target-group on a regular navigation element. The table of contents on this page demonstrates this—as you scroll through sections, the corresponding link highlights automatically.

Conclusion

These new CSS scroll features represent a significant step forward for web development. By moving scroll-driven interactions into CSS, we get:

  • Better performance (no JavaScript scroll listeners)
  • Reduced code complexity
  • Built-in accessibility
  • Smoother animations and transitions

While browser support is still limited to Chromium-based browsers, these features are on track to become web standards. Start experimenting with them today to be ready when they're widely available!

Note: This demo requires Chrome or Edge 135+ to see all features in action.
